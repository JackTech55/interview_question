🔗 Database Relationships

👤

User → Doctor (1:1)

Each doctor user has one doctor profile with professional details, specialization, and credentials

🔗

Doctor ↔ Clinic (M:N)

Many-to-many relationship allowing doctors to work at multiple clinics with different schedules

📅

Patient → Appointments (1:N)

Each patient can have multiple appointments with different doctors and clinics

⭐

Appointment → Review (1:1)

Each completed appointment can have one review and rating from the patient
